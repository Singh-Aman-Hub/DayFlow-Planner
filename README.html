DayFlow Planner

A lightweight web application to plan your day and capture notes. DayFlow helps you understand and allocate your time more intentionally so you can focus on meaningful work.

What this project is

DayFlow Planner is a single-page React application built with Vite. It provides a simple planner where users can compose blocks of time, save plans to local history, and keep a scratchpad for notes. Data is persisted in the browser using localStorage, so the application does not require a backend to retain user data.

Key benefits

  • Helps you gain visibility into your time by planning blocks and viewing saved plans.
  • Lightweight and privacy-first: data is stored locally in the browser.
  • Easy to deploy as a static site (Vite build outputs a dist folder).

Prerequisites

  • Node.js (recommended v18 or later)
  • npm or Yarn

Install and run locally

Clone the repository and install dependencies, then start the Vite dev server:

# clone the repo
git clone https://github.com/Singh-Aman-Hub/DayFlow-Planner.git
cd DayFlow-Planner

# install dependencies (npm)
npm install

# start development server
npm run dev

# open the site at the address printed by Vite (usually http://localhost:5173)

If you prefer Yarn, replace npm install with yarn and use yarn dev.

Build for production

To create an optimized production build:

npm run build

The output directory is dist. Serve files from this directory with any static host or CDN.

Deploying to Render (static site)

Recommended fields when creating a new static site on Render:

  • Branch: master
  • Root Directory: leave empty (project root)
  • Build Command: npm install && npm run build (or yarn install && yarn build)
  • Publish Directory: dist

Environment variables that the project reads at build time (optional):

GEMINI_API_KEY=<your_api_key>

Set GEMINI_API_KEY in the Render dashboard if you use any features that require it. The Vite configuration injects this key at build time into process.env.GEMINI_API_KEY and process.env.API_KEY. Avoid committing secrets to the repository.

Data persistence

All user data (current plan, saved history, notes, and current view) are persisted in the browser's localStorage with the following keys:

  • dayflow_current_plan
  • dayflow_history
  • dayflow_view
  • dayflow_notes

There is no server-side persistence by default. If you clear your browser storage or use a different device, saved plans and notes will not be available unless you implement a backend sync.

How to configure

Configuration is minimal. The main points to adjust are:

  • If you want to change the build output folder, add a build.outDir entry to vite.config.ts. If you do, update the Publish Directory in your host to match.
  • To enable any API integration, add the required environment variables to your host's dashboard before building.

Planned features

  • Server-side syncing so users can persist plans and notes across devices.
  • Authentication and user accounts for personal plan management.
  • Advanced analytics to show time allocation and productivity insights.
  • Shared plans and collaboration features.
  • Integrations with calendar providers and task managers.

These features are not yet implemented. The current implementation focuses on local, private planning with a minimal, distraction-free UI.
